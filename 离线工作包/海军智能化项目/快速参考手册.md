# 离线写作快速参考手册

## 一、文档结构速查

### 技术方案章节模板
```
1. 背景与需求（10%篇幅）
2. 技术体系（20%篇幅）
3. 应用场景（30%篇幅）
4. 关键技术（20%篇幅）
5. 工程实现（10%篇幅）
6. 指标验证（5%篇幅）
7. 实施计划（5%篇幅）
```

### 每个场景的标准结构（8页）
```
- 场景描述（1页）
- 能力需求（1页）
- 技术方案（3页）
- 训练方案（1页）
- 实施路径（1页）
- 预期成果（1页）
```

## 二、常用术语对照

### 中英文对照
- 强化学习 = Reinforcement Learning (RL)
- 多智能体 = Multi-Agent
- 近端策略优化 = Proximal Policy Optimization (PPO)
- 深度Q网络 = Deep Q-Network (DQN)
- 中央训练分散执行 = Centralized Training Decentralized Execution (CTDE)
- 蒙特卡洛树搜索 = Monte Carlo Tree Search (MCTS)
- 长短期记忆网络 = Long Short-Term Memory (LSTM)
- 卷积神经网络 = Convolutional Neural Network (CNN)
- 图神经网络 = Graph Neural Network (GNN)

### 军事术语规范
- 使用"无人平台"而非"机器人"
- 使用"智能体"而非"AI代理"
- 使用"协同作战"而非"团队合作"
- 使用"态势感知"而非"环境感知"
- 使用"火力分配"而非"资源分配"

## 三、数据速查表

### 算力成本速算
```
单日成本 = CPU核数 × 24 × 0.04 + GPU张数 × 24 × 5
周成本 = 单日成本 × 7
项目成本 = 单日成本 × 训练天数
```

### 训练周期估算
- 单体智能体：2-3周
- 多智能体协同：3-4周
- 对抗训练：6-8周
- 大规模集群：7-10周

### 团队配置
- 算法工程师：2-3人
- 系统工程师：1-2人
- 测试工程师：1人
- 项目经理：1人

## 四、写作技巧

### 数据量化原则
❌ 避免："大幅提升"、"显著改善"
✅ 使用："提升40%"、"降低30%"

### 技术深度把握
- 背景章节：浅层介绍，面向领导
- 技术章节：中等深度，面向专家
- 实施章节：操作层面，面向工程师

### 图表使用建议
- 每5页至少1个图表
- 架构图：展示系统结构
- 流程图：展示处理流程
- 对比表：展示性能对比
- 甘特图：展示时间计划

## 五、常见问题速查

### Q1: 如何描述算法选择？
**模板**：
```
针对[场景特点]，选择[算法名称]。
该算法优势在于[优势1]、[优势2]。
相比[其他算法]，[本算法]在[指标]上提升[X]%。
```

### Q2: 如何描述训练过程？
**模板**：
```
训练分为[N]个阶段：
阶段1：[目标]，环境[描述]，成功标准[指标]
阶段2：[目标]，环境[描述]，成功标准[指标]
...
总计训练[X]周，消耗算力[Y]万元。
```

### Q3: 如何描述性能指标？
**模板**：
```
核心指标：
- [指标1]：≥/≤ [数值][单位]
- [指标2]：≥/≤ [数值][单位]
已验证指标：
- [指标1]：实测[数值]（目标[数值]）
```

### Q4: 如何描述实施路径？
**模板**：
```
第一步：[阶段名称]（[时长]）
- [工作内容1]
- [工作内容2]
第二步：[阶段名称]（[时长]）
...
```

1. "首次将[技术]应用于[领域]"
2. "突破[技术难点]，实现[能力]"
3. "提出[方法名称]，解决[问题]"

1. "技术风险：通过[方法]降低风险"

## 八、紧急情况处理

### 场景1：某个技术点不确定
**处理**：
1. 查阅"已验证技术指标.md"
2. 查阅"扩展能力清单.md"
3. 标注"待确认"，继续写作

### 场景2：数据缺失
**处理**：
1. 使用"预期指标"替代
2. 标注数据来源为"仿真预测"
3. 后续补充实测数据

### 场景3：时间不足
**优先级**：
1. 核心章节（2、3、4章）
2. 支撑章节（5、6章）
3. 框架章节（1、7、8章）
