## 2. MARL算法选择

### 2.1 算法对比

| 算法 | 架构 | 适用场景 | 通信需求 |
|------|------|---------|---------|
| QMIX | 中央训练分散执行 | 完全合作、价值分解 | 训练时需要 |
| MADDPG | 中央训练分散执行 | 混合合作竞争 | 训练时需要 |
| MAPPO | 分散训练分散执行 | 完全合作、大规模 | 不需要 |
| CommNet | 通信学习 | 需要信息共享 | 运行时需要 |

### 2.2 推荐方案

**完全合作场景**（编队协同）：QMIX / MAPPO
**部分对抗场景**（红蓝对抗）：MADDPG
**通信受限场景**：MAPPO（分散式）

## 3. 协同架构设计

### 3.1 分层控制架构

```
战略层（全局协调）
    ↓
战术层（编队控制）
    ↓
执行层（单体智能体）
```

**战略层**：
- 任务分配
- 资源调度
- 全局路径规划

**战术层**：
- 编队队形控制
- 协同机动
- 信息融合

**执行层**：
- 单体动作执行
- 局部避障
- 传感器控制

### 3.2 通信机制

**通信内容**：
- 位置、速度、航向
- 目标信息
- 任务状态
- 意图信息

**通信协议**：
- 广播：全局信息（低频）
- 点对点：协同信息（高频）
- 优先级：紧急 > 常规

**通信约束**：
- 带宽限制：[X] Mbps
- 延迟：≤ 200ms
- 丢包率：≤ 5%

## 4. 状态空间设计

### 4.1 局部观测（单智能体）

**自身状态**（维度：20）：
- 位置、速度、姿态
- 装备状态
- 任务信息

**局部环境**（维度：30）：
- 邻近智能体信息（最近5个）
- 局部目标信息
- 局部障碍物

### 4.2 全局状态（中央训练用）

**全局态势**（维度：N×50）：
- 所有智能体状态
- 所有目标信息
- 全局地图信息

### 4.3 通信信息

**接收信息**（维度：K×10）：
- K个邻居的位置、速度
- 共享的目标信息

- 装备控制：传感器、武器

### 5.2 协同动作
- 编队指令：保持/变换队形
- 协同指令：跟随/包围/分散

## 6. 奖励函数设计

### 6.1 个体奖励
```python
individual_reward = task_reward + safety_reward - penalty
```

### 6.2 协同奖励
```python
cooperative_reward = formation_reward    # 编队保持
                   + sync_reward         # 动作同步
                   + coverage_reward     # 区域覆盖
```

### 6.3 全局奖励
```python
global_reward = team_task_reward         # 团队任务完成
              + efficiency_reward        # 整体效率
```

### 6.4 总奖励
```python
total_reward = w1 * individual_reward
             + w2 * cooperative_reward
             + w3 * global_reward
```

**权重建议**：w1=0.3, w2=0.4, w3=0.3

## 7. 训练流程

### 7.1 训练阶段

**阶段1：单体预训练**（1周）
- 每个智能体独立训练基础能力
- 无协同要求

**阶段2：协同训练**（2周）
- 多智能体联合训练
- 学习协同策略

**阶段3：对抗训练**（1周）
- 与对手智能体对抗
- 提升鲁棒性

### 7.2 训练参数

**计算资源**：

- 日均成本：约10000元

**训练超参数**：
- 总训练步数：20M steps
- 并行环境数：256
- 智能体数量：[N] 个
- 更新频率：每4096步

## 8. 编队控制

### 8.1 队形定义

**常用队形**：
- 纵队：适合狭窄水域
- 横队：适合搜索任务
- 楔形：适合进攻任务
- 环形：适合防御任务

### 8.2 队形保持

**虚拟结构法**：
```python
desired_position = leader_position + formation_offset
error = current_position - desired_position
action = PID_controller(error)
```

**行为法**：
- 聚合：向编队中心移动
- 分离：避免碰撞
- 对齐：保持一致航向
