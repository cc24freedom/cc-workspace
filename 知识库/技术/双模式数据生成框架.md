# 双模式数据生成框架

## 核心思路
针对不同项目阶段的数据来源差异，提供冷启动与热启动两种互补的数据生成模式。

## 冷启动模式

**适用场景**：项目初期无历史数据积累

**数据来源**：参数化规则引擎 + 仿真平台

**核心流程**：
1. 参数配置生成（战术类型 × 参数矩阵）
2. 规则引擎执行（生成结构化决策数据）
3. 格式转换（转为仿真想定）
4. 仿真执行（生成过程数据）
5. 数据采集与入库

**优势**：
- 系统性覆盖参数空间，分布均匀
- 结构规整，参数可控
- 可大规模批量生成

**采样策略**：
- 网格采样（系统性覆盖）
- 随机采样（均匀分布）
- 边界增强采样（极端场景）

## 热启动模式

**适用场景**：已获取演习或实战数据

**数据来源**：真实演习数据 / 开源数据

**核心流程**：
1. 数据清洗（格式归一化、异常值处理）
2. 参数扰动增强（在关键参数上施加受控扰动）
3. 扩展样本生成
4. 数据入库

**优势**：
- 保留真实战术知识
- 近真实性高
- 含实战不确定性

## 两种模式对比

| 维度 | 冷启动 | 热启动 |
|------|--------|--------|
| 数据来源 | 规则引擎 | 真实数据 |
| 驱动方式 | 仿真想定 | 数据联合仿真 |
| 数据特征 | 规整可控 | 含噪声 |
| 适用阶段 | 初期 | 后期 |
